# 目标
彻底舍弃之前的“变淡”方案，改为全方位的“深色”风格。不仅加深点亮时的彩色，也将未点亮部分改为不透明的深灰色，使整体视觉效果厚重、深沉。

## 修改计划

### 1. 颜色映射极度加深 (`get_gradient_color`)
将颜色端点调整为低亮度、高饱和的深色系：
- **0% (起始)**: 由浅绿改为 **深草绿** `(0, 120, 0)`。
- **50% (中段)**: 由深绿改为 **墨绿** `(0, 50, 0)`。
- **75% (警戒)**: 由黄色改为 **暗金/褐黄** `(160, 120, 0)`。
- **100% (极限)**: 由红色改为 **深红/酒红** `(140, 0, 0)`。
- 保持插值逻辑不变，仅修改 RGB 数值。

### 2. 未点亮状态改为“实心深灰”
为了配合深色进度条，背景不再使用半透明浅灰，而是实心深灰，增加对比度和质感：
- **初始化循环**:
    - 颜色: `0xDDDDDD` -> `0x444444` (深灰)。
    - 不透明度: `100` -> `255` (完全不透明)。
- **熄灭恢复逻辑 (`left/right_timer_cb`)**:
    - 恢复颜色: `0xDDDDDD` -> `0x444444`。

### 3. 调整呼吸动画 (`cell_light_up` / `cell_light_down`)
由于未点亮状态已改为不透明（255），原有的透明度渐变（80<->255）已不再适用，需调整以保持一致性：
- **点亮动画 (`cell_light_up`)**: 不透明度从 `255` -> `255` (即保持常亮，仅保留缩放呼吸效果，或移除透明度动画)。为最小化改动，将数值改为 `255, 255`。
- **熄灭动画 (`cell_light_down`)**: 不透明度从 `255` -> `255` (保持实心，仅靠颜色变化)。

## 预期效果
- 整个进度条在未操作时呈现为两条深灰色的实心长条。
- 操作时，深邃的绿色/金色/红色在深灰背景上亮起，色彩浓郁，不再有发白或半透明的“轻飘”感。

## 验证
- 检查 `get_gradient_color` 的 RGB 值是否已更新。
- 检查初始化和回调中的未点亮颜色是否为 `0x444444`。
- 检查不透明度是否全程维持 `255`。
